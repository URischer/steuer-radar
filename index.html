ℹ️ Wann stößt dieses Tool an seine Grenzen?

Für einen einzelnen Steuerberater ist dieses Offline-Tool bis mindestens Ende 2027 vollständig ausreichend. Kein Server, keine Kosten, vollständig DSGVO-konform. Ein Backend wird erst dann nötig, wenn:
  • Mehrere Nutzer gleichzeitig bearbeiten — Browser-Tools kennen keinen Merge-Konflikt-Resolver
  • Push-Updates an alle Kanzlei-Rechner — ohne Server kein WebSocket, kein automatischer Abgleich
  • Revisionssicheres Änderungsprotokoll — GoBD-konforme Protokollierung erfordert Backend-DB
  • DATEV- oder Kanzlei-Software-Integration — erfordert API-Schnittstelle serverseitig
  • Automatischer Fristenalarm per Email — Cronjob läuft nur auf einem Server, nicht im Browser
  • Mandanten-Login-Portal — echte Authentifizierung ist ohne Backend nicht sicher realisierbar
💡 Empfehlung: Bei Wachstum auf >2 Nutzer → Migration zu Next.js + Supabase oder ähnlichem Stack sinnvoll (Migrationsaufwand: ca. 2–3 Tage mit KI-Unterstützung).
